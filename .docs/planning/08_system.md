## 8-1. 회원가입 / 로그인

### 지원 방식

| 방식 | 상태 | 설명 |
|------|------|------|
| Google 로그인 | 1차 지원 | OAuth 2.0 기반 소셜 로그인 |
| Kakao 로그인 | 추후 지원 | 카카오 OAuth 연동 |

### 로그인 플로우

```
로그인 버튼 클릭
  → Google OAuth 인증
  → 최초 가입인 경우 → 닉네임 설정 페이지
  → 기존 회원인 경우 → 이전 페이지로 복귀
```

### 닉네임 설정

```
┌─────────────────────────────────┐
│                                 │
│      환영합니다!                 │
│                                 │
│  닉네임을 설정해주세요            │
│                                 │
│  닉네임: [____________]          │
│  (중복 가능, 2~20자)             │
│                                 │
│       [ 시작하기 ]               │
│                                 │
└─────────────────────────────────┘
```

| 항목 | 규칙 |
|------|------|
| 길이 | 2자 이상 20자 이하 |
| 중복 | 허용 (동일 닉네임 가능) |
| 문자 | 한글, 영문, 숫자, 일부 특수문자 |
| 수정 | 설정 후 변경 가능 (프로필 설정에서) |

### 사용자 정보 저장

| 필드 | 출처 | 설명 |
|------|------|------|
| ID | 시스템 생성 | 고유 식별자 |
| 이메일 | OAuth | Google 계정 이메일 |
| 프로필 이미지 | OAuth | Google 프로필 이미지 (기본값) |
| 닉네임 | 사용자 입력 | 직접 설정 |
| 가입일 | 시스템 생성 | 최초 가입 시점 |
| 파트너 여부 | 시스템 | 파트너 가입 상태 |

## 8-2. 반응형 그리드 시스템

### 설계 원칙
- **모바일 퍼스트**: 모바일 기준 카드 배치 후, 웹에서 스케일 확장
- **카드 크기 유지**: 카드 자체 크기는 모바일과 동일하게 유지
- **열 수 자동 조정**: 화면 너비에 따라 열(column) 수만 증가

### 그리드 규격

| 환경 | 화면 너비 | 카드 열 수 | 비고 |
|------|----------|-----------|------|
| 모바일 (기본) | ~480px | 2열 | 기본 레이아웃 |
| 모바일 (넓은) | 481~768px | 3열 | 태블릿 세로 포함 |
| 태블릿 / 소형 웹 | 769~1024px | 4열 | 모바일 대비 2배 |
| 데스크탑 | 1025~1440px | 5열 | |
| 대형 데스크탑 | 1441px~ | 6열 (최대) | 모바일 대비 최대 3배 |

### 구현 방식
- CSS Grid 또는 Flexbox 기반
- `auto-fill` / `auto-fit` + `minmax()` 활용
- 카드 최소 너비 고정, 컨테이너 최대 너비 제한
- 컨테이너 중앙 정렬 (대형 화면에서 좌우 여백)

```
모바일 (2열):
┌──────┐  ┌──────┐
│      │  │      │
└──────┘  └──────┘

데스크탑 (5열):
┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐
│      │ │      │ │      │ │      │ │      │
└──────┘ └──────┘ └──────┘ └──────┘ └──────┘
```

### 그리드 적용 페이지
- 홈 (인플루언서 카드)
- 팔로워 (팔로잉 파트너 카드)
- 이벤트 (이벤트 카드 - 리스트형 또는 그리드형 선택 가능)
